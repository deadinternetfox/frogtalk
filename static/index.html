Set a custom theme for this channel. Users will see this theme when viewing this channel.
🔒 Custom CSS and background images are disabled in private channels. Both features can be used to fingerprint or track invited members. The color pickers above still work for theming.
ℹ️ Custom CSS is sandboxed. Some selectors and properties are stripped on save — show what's blocked
Rules outside the chat area, or that try to reach the rest of the page, are rejected silently. The most common blockers:
  • :root, :host, :scope, :target — anything that escapes the channel scope
  • [data-theme], [data-mode] — platform theme attributes
  • Naked attribute selectors like [class] with no element prefix
  • @import, @namespace, @font-face, @charset, @layer, @container, @property and other @-rules
  • Pseudo-elements that paint outside the matched element: ::column, ::part, ::slotted, ::backdrop, view-transition pseudos, ::-webkit-scrollbar
  • Combinators and grouping that look like injection: >, <, parens, quotes, backslashes, semicolons inside selectors
  • url(), expression(), javascript:, behavior:, -moz-binding, CSS hex / HTML-entity escapes, Unicode whitespace tricks
Allowed: ::before, ::after, ::first-line, ::first-letter, ::marker, ::placeholder, ::selection, :hover, :focus, :active, descendant chains (space-separated tags, classes, IDs), and class-stacking like .a.b.c. Anything with parens (:not(), :nth-child(), calc() in selectors) is dropped — keep selectors simple. Preview before saving — rejected rules show up as a toast.
