Run a FrogTalk Node

Self-host a federated chat node on a fresh Linux VPS — CLI setup wizard, HTTPS, systemd, and sync with the public mesh anchored at frogtalk.xyz.

Prerequisites VPS bootstrap Setup wizard Federation HTTPS Verify Troubleshooting Tor / onion Updates

Full guide (GitHub): docs/NODE_INSTALL.md · Typical path: /opt/frogtalk

Secrets: Never commit .env, SSH passwords, or federation tokens. Use SSH keys for VPS login. Placeholders below: <SSH_USER>, <YOUR_VPS_IP>, <YOUR_DOMAIN>.

Prerequisites

ItemRecommendation
VPS1–2 vCPU, 2–4 GB RAM, 20+ GB disk (scale up for busy boards)
OSDebian 12 / Ubuntu 22.04+ with systemd
Python3.10+ (3.11+ recommended)
DNSA/AAAA for chat.<YOUR_DOMAIN> → VPS IP
Ports22 (SSH), 80/443 (nginx) — keep app port 8080 on localhost only

Install wizard = bash CLI (node/scripts/install.sh), not a browser page. Users sign up at /app after the node is live.

1) Bootstrap the VPS

# SSH (prefer keys — do not store passwords in git)
ssh-copy-id -i ~/.ssh/id_ed25519.pub <SSH_USER>@<YOUR_VPS_IP>
ssh <SSH_USER>@<YOUR_VPS_IP>

# Packages
sudo apt update
sudo apt install -y git python3 python3-venv python3-pip curl nginx \
  certbot python3-certbot-nginx ufw

# Optional dedicated user (matches frogtalk.service default)
sudo adduser --disabled-password --gecos "" deploy
sudo mkdir -p /opt/frogtalk && sudo chown deploy:deploy /opt/frogtalk

# Firewall
sudo ufw allow OpenSSH
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable

2) Clone repo + run setup wizard

cd /opt
git clone https://github.com/deadinternetfox/frogtalk.git
cd frogtalk

# Interactive menu: setup · federation · systemd · updates
bash node/scripts/install.sh

# Non-interactive (fresh VPS)
bash node/scripts/install.sh setup -y --install-dir /opt/frogtalk
bash node/scripts/install.sh federation -y --install-dir /opt/frogtalk
bash node/scripts/install.sh systemd -y --install-dir /opt/frogtalk

The wizard creates venv/, writes .env, and symlinks node/data, node/.env, node/secrets to the install root. A real node/data folder (not a symlink) causes an empty database — the wizard and federation script repair this.

Admin: user admin is created on first boot. If ADMIN_PASSWORD is empty, a one-time password is logged to journalctl -u frogtalk — log in and rotate immediately.

# Dev run without systemd
cd /opt/frogtalk && source venv/bin/activate && cd node && python main.py
# → http://127.0.0.1:8080

3) Federation — join the mesh

bash node/scripts/node_federation_join.sh --install-dir /opt/frogtalk -y \
  --public-url https://chat.<YOUR_DOMAIN>

# Re-sync after changing URL or major upgrades
bash node/scripts/install.sh federation -y

Pulls the official directory, imports peers into SQLite, and pins Ed25519 keys from each peer’s /api/network/status (the directory JSON does not ship pubkeys).

Official production anchors (also used as CLI fallback if directory HTTP fails):

NameRole
FrogTalk Mainhttps://frogtalk.xyz
FrogTalk Tor MirrorOnion mirror — see network list in the app

Essential .env keys (wizard sets most of these):

FROGTALK_FEDERATION_ENABLED=1
FROGTALK_FEDERATION_REQUIRE_SIGS=1
FROGTALK_OFFICIAL_DIRECTORY_URL=https://frogtalk.xyz/api/network/servers
PUBLIC_URL=https://chat.<YOUR_DOMAIN>
ALLOWED_ORIGINS=https://chat.<YOUR_DOMAIN>
FROGTALK_AUTO_UPDATE_ENABLED=0

4) nginx + HTTPS

Terminate TLS at nginx; proxy to uvicorn on 127.0.0.1:8080 (default PORT=8080 in .env).

# /etc/nginx/sites-available/frogtalk
server {
    listen 80;
    server_name chat.<YOUR_DOMAIN>;
    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

sudo ln -sf /etc/nginx/sites-available/frogtalk /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx
sudo certbot --nginx -d chat.<YOUR_DOMAIN>

Template reference: node/deploy/nginx.conf. If upstream uses port 8000, either set PORT=8000 or change proxy_pass — board internal API defaults assume the same port.

5) Verify

bash node/scripts/install.sh status
curl -sS https://chat.<YOUR_DOMAIN>/api/ping
curl -sS https://chat.<YOUR_DOMAIN>/api/network/status | python3 -m json.tool
journalctl -u frogtalk -n 40 --no-pager

Open https://chat.<YOUR_DOMAIN>/app, create a test account, join a room. In Settings → Network, probe peers and compare build hash with frogtalk.xyz before trusting federation.

Backup: stop the service, archive .env, data/frogtalk.db, data/uploads, and secrets/.

Tor / onion (optional)

FROGTALK_TOR_ENABLED=1
FROGTALK_ONION_URL=http://yourhiddenservice.onion
# Clearnet hub fetching .onion boards:
FROGTALK_TOR_SOCKS_PROXY=socks5://127.0.0.1:9050

Clients should open http://….onion/app. Clearnet hubs need Tor SOCKS to reach onion peers. Voice/video in Tor Browser is limited — use the mobile app or a normal browser window.

bash node/scripts/node_federation_join.sh --install-dir /opt/frogtalk -y \
  --onion-url http://yourhiddenservice.onion --public-url ""

Updates & signed releases

bash node/scripts/install.sh update
bash node/scripts/install.sh update-apply -y

FROGTALK_UPDATE_FEED_URL defaults to frogtalk.xyz. Set FROGTALK_RELEASE_SIGNERS (Ed25519 hex) before enabling FROGTALK_AUTO_UPDATE_ENABLED=1.

Cross-node calls: FROGTALK_FEDERATION_CALLS_ENABLED=1 + coturn — see docs/FEDERATED_CALLS.md in the repo.
